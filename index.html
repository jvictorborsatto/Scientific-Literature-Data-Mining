Como funciona

📋

Lista de Objetos

Cadastre compostos, espécies, genes ou qualquer entidade de interesse, com categorias, subcategorias e sinônimos. Importe de CSV com um clique.

🔍

Mineração de Artigos

Adicione artigos em PDF ou cole o texto manualmente. O scanner detecta automaticamente quais objetos aparecem em cada artigo.

📊

Visualizações Interativas

Gere gráficos de barras, linhas, dispersão, mapas de calor e pizza diretamente dos seus dados. Exportáveis em alta resolução.

🗃️

Compilador de Dados

Organize dados quantitativos de revisões sistemáticas (LC50, EC50, IC50…). Padronize nomes de parâmetros e monte tabelas de síntese.

💾

Projetos Portáteis

Tudo salvo em um único arquivo .sldm (SQLite). Fácil de compartilhar, fazer backup ou migrar entre computadores.

📝

Editor CSV Integrado

Visualize e edite tabelas de dados diretamente no app, sem precisar abrir planilhas externas. Ideal para ajustes rápidos.

Fluxo típico de uso

1

Crie um projeto

Abra o SLDM e crie um novo projeto. Um arquivo .sldm será gerado — ele guarda tudo.

2

Cadastre seus objetos de interesse

Na aba Object List, adicione compostos, espécies ou genes. Inclua sinônimos para melhorar a detecção automática.

3

Importe seus artigos

Na aba Article Mining, adicione PDFs ou cole os textos. Use o botão Scan para detectar automaticamente quais objetos aparecem.

4

Compile os dados quantitativos

No Data Compiler, insira os valores extraídos dos artigos (doses, concentrações, etc.) e monte tabelas de revisão.

5

Gere visualizações

Na aba Visualization, escolha o tipo de gráfico, configure os eixos e exporte a imagem pronta para o seu artigo.

Importando dados via CSV

O SLDM aceita importação em formato .csv para agilizar o cadastro em massa de objetos e dados de revisão. Abaixo estão as especificações de cada arquivo suportado.

📋
CSV de Objetos
File → Import Objects CSV · ou aba Object List

Use este arquivo para cadastrar os objetos que o SLDM irá rastrear nos artigos. Cada linha representa um objeto distinto. Os campos object e category são obrigatórios; os demais são opcionais, mas fortemente recomendados para melhorar a precisão da detecção automática.

cabeçalho esperado
object, category, subcategory, synonyms, notes
object obrigatório
Nome principal do objeto. Será o identificador usado em todo o sistema.
category obrigatório
Classificação ampla do objeto. Usada para filtrar e agrupar nos módulos de análise.
subcategory
Subdivisão dentro da categoria. Permite granularidade adicional na organização.
synonyms
Nomes alternativos, separados por vírgula dentro de aspas duplas. Ampliam o reconhecimento nos textos.
notes
Campo livre para anotações internas. Não afeta a mineração.

⚠ Sinônimos que contêm vírgula devem estar entre aspas duplas no CSV: "nome A, nome B"

🗃️
CSV de Dados de Revisão
Aba Data Compiler

Use este arquivo para importar dados quantitativos extraídos dos artigos — como valores de toxicidade, doses, concentrações ou qualquer parâmetro numérico da sua revisão. Cada linha representa uma medição de um parâmetro para um objeto em um artigo específico.

cabeçalho esperado
object, parameter, value, unit, species, article, notes
object obrigatório
Nome do objeto conforme cadastrado na Object List. Deve haver correspondência exata.
parameter obrigatório
Nome do parâmetro medido. O Data Compiler permite padronizar variantes de um mesmo parâmetro.
value obrigatório
Valor numérico da medição. Use ponto como separador decimal.
unit
Unidade da medição. Recomendado para consistência nas visualizações.
species
Organismo ou modelo de estudo. Útil para filtros e agrupamentos nos gráficos.
article
Referência ao artigo de origem. Permite rastrear de onde cada valor foi extraído.
notes
Campo livre para observações sobre a medição ou condições experimentais.

Interface gráfica

O SLDM usa PyQt5 como framework de UI, com uma janela principal baseada em abas (QTabWidget). O tema visual é aplicado globalmente via QSS (Qt Style Sheets), garantindo aparência consistente.

PyQt5 QTabWidget QSS Qt5

Banco de dados

Todos os dados — artigos, objetos, parâmetros e revisões — são armazenados em um banco SQLite embutido no arquivo .sldm. A camada de acesso (database.py) abstrai todas as queries.

SQLite sqlite3 arquivo único

Extração de texto & mineração

PDFs são processados via PyMuPDF (fitz), que extrai o texto bruto de cada página. O scanner percorre o texto buscando os objetos cadastrados e seus sinônimos usando correspondência por regex normalizada.

PyMuPDF fitz regex NLP básico

Visualização de dados

Os gráficos são renderizados com Matplotlib embutido no PyQt via FigureCanvasQTAgg. O Pandas gerencia os DataFrames intermediários usados para agregar e pivotear os dados antes da plotagem.

Matplotlib FigureCanvasQTAgg Pandas DataFrame

Sobre o projeto

SLDM

O SLDM nasceu da necessidade prática de organizar grandes volumes de literatura científica em revisões sistemáticas de toxicologia e ecotoxicologia, onde identificar manualmente a presença de centenas de compostos em dezenas de artigos é um processo lento e sujeito a erros.

O software foi desenvolvido integralmente em Python, com foco em usabilidade para pesquisadores sem formação técnica em programação. A interface foi projetada para refletir o fluxo natural de uma revisão bibliográfica: cadastrar → minerar → compilar → visualizar.

Distribuído como executável independente, sem necessidade de configuração de ambiente, o SLDM pode ser usado imediatamente em qualquer computador Windows — e potencialmente em Linux e macOS.
