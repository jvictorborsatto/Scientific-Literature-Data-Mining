How it works

Como funciona

📋

Object List

Lista de Objetos

Register compounds, species, genes or any entity of interest, with categories, subcategories and synonyms. Import from CSV in one click.

Cadastre compostos, espécies, genes ou qualquer entidade de interesse, com categorias, subcategorias e sinônimos. Importe de CSV com um clique.

🔍

Article Mining

Mineração de Artigos

Add articles as PDF or paste text manually. The scanner automatically detects which objects appear in each article.

Adicione artigos em PDF ou cole o texto manualmente. O scanner detecta automaticamente quais objetos aparecem em cada artigo.

📊

Interactive Visualizations

Visualizações Interativas

Generate bar charts, line plots, scatter plots, heatmaps and pie charts directly from your data. Exportable in high resolution.

Gere gráficos de barras, linhas, dispersão, mapas de calor e pizza diretamente dos seus dados. Exportáveis em alta resolução.

🗃️

Data Compiler

Compilador de Dados

Organize quantitative data from systematic reviews (LC50, EC50, IC50…). Standardize parameter names and build synthesis tables.

Organize dados quantitativos de revisões sistemáticas (LC50, EC50, IC50…). Padronize nomes de parâmetros e monte tabelas de síntese.

💾

Portable Projects

Projetos Portáteis

Everything saved in a single .sldm file (SQLite). Easy to share, back up or migrate between computers.

Tudo salvo em um único arquivo .sldm (SQLite). Fácil de compartilhar, fazer backup ou migrar entre computadores.

📝

Built-in CSV Editor

Editor CSV Integrado

View and edit data tables directly inside the app, without opening external spreadsheets. Ideal for quick adjustments.

Visualize e edite tabelas de dados diretamente no app, sem precisar abrir planilhas externas. Ideal para ajustes rápidos.

Typical workflow

Fluxo típico de uso

1

Create a project

Crie um projeto

Open SLDM and create a new project. A .sldm file will be created — it stores everything.

Abra o SLDM e crie um novo projeto. Um arquivo .sldm será gerado — ele guarda tudo.

2

Register your objects of interest

Cadastre seus objetos de interesse

In the Object List tab, add compounds, species or genes. Include synonyms to improve automatic detection.

Na aba Object List, adicione compostos, espécies ou genes. Inclua sinônimos para melhorar a detecção automática.

3

Import your articles

Importe seus artigos

In the Article Mining tab, add PDFs or paste texts. Use the Scan button to automatically detect which objects appear.

Na aba Article Mining, adicione PDFs ou cole os textos. Use o botão Scan para detectar automaticamente quais objetos aparecem.

4

Compile quantitative data

Compile os dados quantitativos

In Data Compiler, enter values extracted from articles (doses, concentrations, etc.) and build review tables.

No Data Compiler, insira os valores extraídos dos artigos (doses, concentrações, etc.) e monte tabelas de revisão.

5

Generate visualizations

Gere visualizações

In the Visualization tab, choose the chart type, configure axes and export the image ready for your paper.

Na aba Visualization, escolha o tipo de gráfico, configure os eixos e exporte a imagem pronta para o seu artigo.

Importing data via CSV

Importando dados via CSV

SLDM supports .csv import to speed up bulk registration of objects and review data. Below are the specifications for each supported file.

O SLDM aceita importação em formato .csv para agilizar o cadastro em massa de objetos e dados de revisão. Abaixo estão as especificações de cada arquivo suportado.

Objects CSVCSV de Objetos
File → Import Objects CSV

Use this file to register the objects SLDM will track in articles. Each row is a distinct object. The object and category fields are required; others are optional but strongly recommended to improve detection accuracy.

Use este arquivo para cadastrar os objetos que o SLDM irá rastrear nos artigos. Cada linha representa um objeto distinto. Os campos object e category são obrigatórios; os demais são opcionais, mas fortemente recomendados para melhorar a precisão da detecção automática.

Graphical interface

Interface gráfica

SLDM uses PyQt5 as its UI framework, with a tab-based main window (QTabWidget). The visual theme is applied globally via QSS (Qt Style Sheets) for a consistent look.

O SLDM usa PyQt5 como framework de UI, com uma janela principal baseada em abas (QTabWidget). O tema visual é aplicado globalmente via QSS, garantindo aparência consistente.

PyQt5QTabWidgetQSSQt5

Database

Banco de dados

All data — articles, objects, parameters and reviews — is stored in a SQLite database embedded in the .sldm file. The access layer (database.py) abstracts all queries.

Todos os dados — artigos, objetos, parâmetros e revisões — são armazenados em um banco SQLite embutido no arquivo .sldm. A camada de acesso (database.py) abstrai todas as queries.

SQLitesqlite3single filearquivo único

Text extraction & mining

Extração de texto & mineração

PDFs are processed via PyMuPDF (fitz), extracting raw text from each page. The scanner searches the text for registered objects and synonyms using normalised regex matching.

PDFs são processados via PyMuPDF (fitz), extraindo o texto bruto de cada página. O scanner busca os objetos cadastrados e seus sinônimos usando correspondência por regex normalizada.

PyMuPDFfitzregexbasic NLPNLP básico

Data visualization

Visualização de dados

Charts are rendered with Matplotlib embedded in PyQt via FigureCanvasQTAgg. Pandas manages intermediate DataFrames used to aggregate and pivot data before plotting.

Os gráficos são renderizados com Matplotlib embutido via FigureCanvasQTAgg. O Pandas gerencia os DataFrames intermediários usados para agregar e pivotar os dados.

MatplotlibFigureCanvasQTAggPandasDataFrame

About the project

Sobre o projeto

SLDM

SLDM was born from the practical need to organize large volumes of scientific literature in systematic reviews of toxicology and ecotoxicology, where manually identifying the presence of hundreds of compounds across dozens of articles is a slow and error-prone process.

O SLDM nasceu da necessidade prática de organizar grandes volumes de literatura científica em revisões sistemáticas de toxicologia e ecotoxicologia, onde identificar manualmente a presença de centenas de compostos em dezenas de artigos é um processo lento e sujeito a erros.

The software was developed entirely in Python, with a focus on usability for researchers without a technical programming background. The interface was designed to reflect the natural flow of a bibliographic review: register → mine → compile → visualize.

O software foi desenvolvido integralmente em Python, com foco em usabilidade para pesquisadores sem formação técnica em programação. A interface foi projetada para refletir o fluxo natural de uma revisão bibliográfica: cadastrar → minerar → compilar → visualizar.

Distributed as a standalone executable requiring no environment setup, SLDM can be used immediately on any Windows computer.

Distribuído como executável independente, sem necessidade de configuração de ambiente, o SLDM pode ser usado imediatamente em qualquer computador Windows.
